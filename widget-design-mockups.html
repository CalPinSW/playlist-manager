Implementation Plan

📦 Phase 1 — Native Scaffold

  • 1Install @bndkt/react-native-widget-extension + add plugin to app.json
  • 2Create App Group in Apple Dev Portal (e.g. group.com.you.playlistmanager) — one-time
  • 3Add App Group entitlement to both main app and widget extension targets in app.json
  • 4EAS Build to compile native targets — required before anything works

🗃️ Phase 2 — Data Bridge (JS → Widget)

  • 1On every fetchNowPlaying() poll tick (already 10s), write JSON to App Group UserDefaults
  • 2Payload: albumId, albumName, artistName, imageUrl (local path!), rating
  • 3Download and cache album art to App Group container — widgets cannot load http:// URLs
  • 4Call WidgetCenter.shared.reloadAllTimelines() after each write (bridged to JS via module)

⌚ Phase 3 — SwiftUI Widget Views

  • 1Read App Group UserDefaults in TimelineProvider, decode JSON to NowPlayingEntry
  • 2Implement .systemSmall (S1) and .systemMedium (M1) views in SwiftUI
  • 3Timeline policy: .never (app drives all refreshes via reloadAllTimelines)
  • 4Tap on widget body → deep-links to playlistmanager://album/{albumId}

⚡ Phase 4 — Interactive Rating (iOS 17+)

  • 1Define SetRatingIntent: AppIntent with albumId: String + rating: Int params
  • 2In perform(): POST to Vercel /api/ratings using stored auth token, write new rating to UserDefaults, call reloadAllTimelines()
  • 3Wrap each star chip in SwiftUI Button(intent: SetRatingIntent(rating: n))
  • 4On older iOS: chips hidden, tap opens app to album detail — graceful fallback

🔔 Phase 5 — Album Change Detection

  • 1Foreground (free): existing 10s poll already fires reloadTimelines via Phase 2 — widget updates in real time while app is open
  • 2Background (device config): enable Background App Refresh in iOS Settings → app wakes every ~15 min, polls now-playing, updates widget
  • 3Silent push (best): Vercel backend detects album change during history sync, sends APNs silent push → app wakes, polls, updates widget within ~30s of change
  • 4Silent push requires APNs cert + remote-notification background mode in app.json

🔑 Auth in Swift (Phase 4 dependency)

  • 1Widget extension runs in a separate process — it cannot read Expo SecureStore directly
  • 2Store the Vercel JWT in App Group Keychain (shared across app + extension) rather than SecureStore
  • 3JS side: on login/token refresh, write new token to App Group Keychain via a native module
  • 4Swift side: SetRatingIntent.perform() reads token from App Group Keychain, calls API
Recommended build order: Phases 1–3 ship a read-only widget that shows now-playing with a displayed (non-tappable) rating. That's already useful and validates the whole pipeline. Phase 4 adds the interactive rating — the payoff feature — but depends on Phase 6 (auth in Swift) being solved first. Phase 5 step 3 (silent push) is optional but makes the "updates when album changes" promise feel real without the app being open.

The interactive rating chips are the standout feature — no other music app does this. You tap a star from your Home Screen and the rating is saved. That's the whole value proposition of this widget.
