## Key Interactions to Verify

### Playlist filter regex
- `NEW_ALBUMS_REGEX` matches `"New Albums 04/04/26"` ✓
- `NEW_ALBUMS_REGEX` rejects `"Best Albums 04/04/26"`, `"New Albums 4/4/26"` (no leading zero), bare `"Albums"`
- `ALL_ALBUMS_REGEX` matches anything with `"Albums"` in the name

### Sync task: track → album reconciliation
- 50 tracks from `recently_played`, all from same album in a New Albums playlist → correct `last_track_index`
- Track NOT in any New Albums playlist → silently skipped, no error thrown
- Existing progress at index 7, new sync data at index 5 → stays at 7 (no regression)
- Existing progress at index 5, new sync data at index 7 → advances to 7
- Empty `recently_played` response → no DB writes, no error

### Sync task: cursor management
- First run (no cursor in sync_log) → calls recently_played without `after` param
- After run, `sync_log.last_played_at` = `played_at` of most recent track
- Second run → passes stored cursor as `after` param

### Token refresh atomicity
- Successful Spotify token refresh → `access_token` + `refresh_token` updated in single transaction
- Prisma transaction write fails → original `refresh_token` NOT overwritten (rollback)
- Missing refresh token → throws `Error('No refresh token found for user')`

### Prisma upsert semantics
- `listening_progress` ON CONFLICT (user_id, album_id, playlist_id) → updates only when new `last_track_index` > existing
- `sync_log` upsert → updates `last_played_at` on every run

## Edge Cases
- Playlist name `"New Albums 4/4/26"` (single-digit day/month) → NOT matched by regex (document this explicitly, use zero-padded dates)
- User has zero playlists matching `NEW_ALBUMS_REGEX` → task completes cleanly, no error
- Spotify 429 on `recently_played` call → Trigger.dev retry handles it (test that retry doesn't break cursor)
- `recently_played` returns tracks with `null` album_id (rare Spotify edge case) → gracefully skip

## Critical Paths
- Listen on phone overnight → open app in the morning → progress reflects where you left off
- Web app displays `listening_progress` correctly after a sync run (Cypress E2E, after schema migration)

## Framework
Vitest for unit tests. Cypress for E2E. Mock Spotify SDK and Prisma in unit tests
(same pattern as `tests/app/api/shows.test.js`).
